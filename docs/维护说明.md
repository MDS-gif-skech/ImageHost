# 维护说明

## 核心组件

```text
GitHub Pages
  负责公开访问图片、上传页和索引。

Cloudflare Worker
  负责接收上传请求，并用 GitHub API 写入仓库。

GitHub 仓库
  负责长期保存图片和索引。
```

## 关键文件

```text
upload.html
  前端上传页面。

worker/src/worker.js
  Cloudflare Worker 主程序。

worker/wrangler.toml
  Worker 配置。

data/images.json
data/images.csv
  图片索引。
```

## Worker 环境变量

普通变量：

| 名称 | 示例 | 说明 |
| --- | --- | --- |
| `GITHUB_OWNER` | `MDS-gif-skech` | GitHub 用户名 |
| `GITHUB_REPO` | `ImageHost` | 图床仓库名 |
| `GITHUB_BRANCH` | `main` | 写入分支 |
| `PUBLIC_BASE_URL` | `https://mds-gif-skech.github.io/ImageHost` | 图片公开 URL 前缀 |
| `MAX_IMAGE_BYTES` | `10485760` | 单张图片最大大小 |
| `ALLOW_ORIGIN` | `*` | 允许跨域来源 |

Secret 变量：

| 名称 | 必填 | 说明 |
| --- | --- | --- |
| `GITHUB_TOKEN` | 是 | GitHub fine-grained token |
| `UPLOAD_SECRET` | 否 | 上传密码 |

## GitHub Token 权限

建议使用 fine-grained personal access token。

Repository access 只选择：

```text
MDS-gif-skech/ImageHost
```

Repository permissions 至少需要：

```text
Contents: Read and write
Metadata: Read-only
```

## 常见问题

### 上传成功但 Pages 打不开新图片

GitHub Pages 可能有几十秒缓存，稍等后刷新即可。

### 上传返回 GitHub API 401

检查 `GITHUB_TOKEN` 是否填错，或 token 是否过期。

### 上传返回 GitHub API 403

检查 token 是否有 `Contents: Read and write` 权限。

### 上传返回 Invalid upload secret

上传页填写的密码和 Worker 的 `UPLOAD_SECRET` 不一致。
