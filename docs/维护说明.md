# 维护说明

## 仓库结构

```text
ImageHost/
  images/                 # 图片文件
  data/
    images.json           # 图片索引 JSON
    images.csv            # 图片索引 CSV
  scripts/
    process_issue_upload.py
  .github/
    workflows/
      upload-from-issue.yml
    ISSUE_TEMPLATE/
      upload-image.yml
  index.html
  README.md
```

## 自动化流程

1. 用户创建 issue 或 issue 评论。
2. workflow `Upload images from issues` 触发。
3. 检查上传者权限。
4. 解析正文中的 GitHub 图片附件。
5. 下载图片并校验格式。
6. 保存到 `images/YYYY/MM/`。
7. 更新 `data/images.json` 和 `data/images.csv`。
8. commit 并 push 回仓库。
9. 在 issue 下回复最终 URL。

## 仓库变量

建议设置：

| 变量 | 说明 | 示例 |
| --- | --- | --- |
| `PUBLIC_BASE_URL` | 图床公开访问根 URL | `https://Ayayadaze.github.io/ImageHost` |
| `MAX_IMAGE_BYTES` | 单张图片最大字节数 | `10485760` |

## 绑定自定义域名

如果以后绑定域名，例如：

```text
https://img.example.com
```

需要做三件事：

1. 在 GitHub Pages 里设置 Custom domain。
2. 在 DNS 里添加 CNAME。
3. 把仓库变量 `PUBLIC_BASE_URL` 改成 `https://img.example.com`。

这样后续新生成的 URL 就会使用你的自定义域名。

## 安全建议

1. 当前是开放上传，方便使用，但如果出现滥用，建议改回白名单或 collaborator 限制。
2. 不建议允许 SVG。
3. 如果发现滥用，先关闭 Issues，或清空 `ALLOWED_UPLOADERS`。
4. 图片公开后不要上传隐私、证件、合同等敏感内容。
