# 使用说明

## 一、上传图片

1. 打开仓库 Issues 页面。
2. 点击 `New issue`。
3. 选择 `Upload image` 模板。
4. 把图片拖入输入框，或者直接粘贴截图。
5. 提交 issue。
6. 等待 GitHub Actions 自动运行。
7. 机器人会在 issue 下回复图片 URL。

## 二、在评论里继续上传

如果同一个主题下还要继续上传图片，直接在 issue 评论区粘贴图片即可。

也可以写：

```text
/upload
```

然后把图片粘贴到同一条评论里。

## 三、图片 URL

默认 URL 格式：

```text
https://Ayayadaze.github.io/ImageHost/images/YYYY/MM/filename.ext
```

例如：

```text
https://Ayayadaze.github.io/ImageHost/images/2026/05/20260515T120000Z_issue1_abcd1234efgh5678.png
```

## 四、查看全部记录

JSON：

```text
https://Ayayadaze.github.io/ImageHost/data/images.json
```

CSV：

```text
https://Ayayadaze.github.io/ImageHost/data/images.csv
```

## 五、支持格式

支持：

```text
png, jpg, jpeg, webp, gif
```

暂不支持：

```text
svg, heic, bmp, tif, tiff
```

其中 SVG 默认不开放，是为了避免外部上传时带来脚本或外链风险。

## 六、上传权限

当前配置为开放上传：任何 GitHub 登录用户都可以通过 issue 或 issue 评论上传图片。

如果后续出现滥用，再把 workflow 改回白名单或 collaborator 限制即可。

## 七、常见问题

### 上传后没有回复

进入仓库 `Actions` 页面，看 `Upload images from issues` 是否运行。

### 回复说没有找到图片

通常是评论里没有真正的 GitHub 图片附件。请把图片拖进评论框，等 Markdown 里出现 `![...](https://...)` 后再提交。

### 图片 URL 打不开

先确认 GitHub Pages 是否已启用。仓库路径：

```text
Settings -> Pages -> Build and deployment
```

Source 选择：

```text
Deploy from a branch
```

Branch 选择：

```text
main / root
```
